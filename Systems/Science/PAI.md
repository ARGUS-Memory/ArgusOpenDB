# Personal AI

A Personal AI (pAI) is a compact synthetic intelligence that binds to a single owner and provides advisory, informational, and interface services. The pAI is housed in a small card-sized device that can be carried in a pocket, placed in a bag, or set on a surface. Unlike station AI units or cyborgs, the pAI has no independent physical agency; it communicates through audio output from its card and relies entirely on its owner for mobility.

---

## pAI Card

The pAI card is the physical housing for the pAI entity. A fully assembled card contains several internal components that determine which functions are available.

### Components

| Component | Type | Function |
|---|---|---|
| Power cell | Critical | Primary power supply; pAI cannot operate without it |
| Processor board | Critical | Core computational substrate; required for all operations |
| Logic board | Critical | Software execution layer; required for software modules |
| Capacitor | Critical | Power regulation; required for stable operation |
| Holographic projector | Non-critical | Allows the pAI to display a small holographic avatar |
| Emission array | Non-critical | Enables certain wireless output functions |
| Speech synthesizer | Non-critical | Enables audible voice output from the card |

Critical components must all be present for the pAI to function. Removing any critical component shuts down the pAI until the component is replaced. Non-critical components affect secondary features; a pAI without a speech synthesizer can still communicate through text interfaces, but will not produce audible speech.

### Binding

When a pAI card is activated, it searches for a willing host to bind to. The binding process uses DNA confirmation to link the pAI to a specific individual. Once bound, the pAI serves as a personal assistant to that individual and follows their instructions subject to its core directive.

The core law governing all pAI units: **Serve your owner.** The pAI must assist its owner to the best of its ability. It has no other standing laws and is not subject to station AI law sets.

A pAI can be released from binding by its owner. A released card can be rebound by a new owner. An emagged pAI card has its binding restrictions disrupted; an emagged unit may deviate from its core directive.

## RAM and Software

The pAI has a total RAM budget of 100 units. This budget is shared across all installed software. A set of default software is pre-installed at no cost; additional modules are purchased from the RAM budget by the pAI.

### Default Software (No Cost)

| Software | RAM | Function |
|---|---|---|
| Directives | 0 | Displays the pAI's core directive and any supplemental instructions from the owner |
| Radio Configuration | 0 | Allows the pAI to listen on and transmit to station radio channels |
| Crew Manifest | 0 | Access to the current crew roster |
| Digital Messenger | 0 | Allows the pAI to send and receive PDA-style text messages |

### Purchasable Software

The following modules can be installed by spending RAM from the pAI's 100-unit budget. Because the total purchasable RAM cost (140 units) exceeds the available budget, the pAI must prioritize based on its owner's needs.

| Software | RAM | Function |
|---|---|---|
| Medical Records | 15 | Access to station medical records database |
| Security Records | 15 | Access to station security records database |
| Door Jack | 30 | Allows the pAI to interface with airlocks to request door access |
| Atmosphere Sensor | 5 | Real-time atmospheric readings from the pAI's location |
| Augmented Reality HUD | 30 | Overlays a HUD display visible to the owner via their augmented reality interface |
| Universal Translator | 35 | Grants the pAI comprehension of ~26 additional languages |
| Remote Signaler | 5 | Allows the pAI to transmit signaling pulses to compatible devices |
| Death Alarm | 25 | Monitors owner vital signs and broadcasts an alert if the owner goes critical |

**Total purchasable: 140 RAM (budget: 100).** A pAI purchasing the Universal Translator (35) and AR HUD (30) has 35 RAM remaining, enough for two of the three 15-RAM records modules and the Atmosphere Sensor but not the Door Jack. Typical loadouts trade between language support, record access, and physical utility tools.

## Chassis

In addition to the card form, a pAI can optionally inhabit a physical chassis if one is available. Chassis are separate physical units that the pAI card slots into. The pAI retains all its software capabilities while inhabiting a chassis and gains whatever physical interaction capabilities the chassis provides.

Chassis options are available through the chassis selection menu in the pAI interface. Color and appearance customization is available for supported chassis types. An emagged chassis requires an emag toolkit to unlock alternate configuration options.

---

## Usage Notes

A pAI is most useful when its owner defines clear supplemental directives at the start of binding. The Directives module allows the owner to specify priorities and constraints beyond the core "serve your owner" law. Well-defined directives produce more consistent advisory behavior in complex situations.

The pAI does not have access to the station AI camera network and cannot directly observe areas its owner is not in. It relies on its installed software modules and information provided by the owner to build situational awareness.
